## 1. 프로젝트 개요

### 1.1 캠페인 컨셉

보이스피싱범의 다양한 유도 문장들이 모여 거대한 회오리(싱크홀) 형태를 이루고, 사용자를 부지불식간에 빨아들이는 참여형 시뮬레이션 이벤트다. 사용자는 실제 보이스피싱 음성을 듣고 상황을 판단하며, 각 선택의 결과를 즉각적으로 체험한다.

**핵심 메시지:** "난 절대로 보이스피싱에 속지 않아!" — 과연 그럴까요?

### 1.2 핵심 목표

- 몰입형 시뮬레이션을 통한 보이스피싱 예방 교육 효과 극대화
- 설치 불필요한 웹 기반(HTML/CSS/JS) 구현으로 접근성 확보
- 모바일 우선(Mobile First) 설계로 SNS 바이럴 유도
- 경품 응모 연계를 통한 참여 완주율 향상
- 금감원 공식 교육 콘텐츠(카드뉴스, 영상) 자연스러운 노출

### 1.3 광고주(금감원) 주요 피드백 반영 사항

| 피드백 항목           | 적용 방향                                             |
| --------------------- | ----------------------------------------------------- |
| 게임 진행 시간 최소화 | Stage를 3개로 압축, 스테이지당 30~60초 내 완료 목표   |
| 참여자 몰입감 극대화  | 실제 음성 재생 + 타이핑 효과 + 싱크홀 애니메이션 연동 |
| 이벤트 URL 간결화     | 보이스피싱크홀.com 또는 간단 영문명 사용 권장         |
| 선택 후 이탈 방지     | A/B 어떤 선택이든 탈락 없이 다음 Stage로 진행         |
| 교육 콘텐츠 연동      | 최종 화면에 카드뉴스/영상/10계명 아이콘 배치          |
| 경품 응모 연계        | 최종 리포트 화면에서 SNS 공유 URL 및 응모 폼 제공     |

### 2.2 기술 스택

| 구분       | 선택 기술                        | 선택 이유                                      |
| ---------- | -------------------------------- | ---------------------------------------------- |
| 언어       | Vanilla JS (ES6+)                | 번들러 없이 단일 파일 납품 가능, 외부 의존성 0 |
| 스타일     | CSS3 + Custom Properties         | JS 없이 테마 전환, 애니메이션 GPU 최적화       |
| 애니메이션 | CSS + Web Animations API         | SVG/Canvas 없이도 싱크홀 효과 구현 가능        |
| 오디오     | Web Audio API + HTMLAudioElement | 브라우저 자동재생 정책 우회, iOS 호환          |
| 폰트       | Google Fonts (Noto Sans KR)      | 한글 최적화, 별도 라이선스 불필요              |
| 빌드 도구  | 없음 (Zero Build)                | 서버 업로드 즉시 실행, 의존성 0                |

## 5. 핵심 모듈 상세 구현 스펙

### 5.1 StageManager.js — 스테이지 전환 엔진

- **역할:** 전체 게임 상태(state) 관리 및 화면 전환 오케스트레이션
- **핵심 메서드:**
  - `StageManager.init()` — 초기화, Stage 1 데이터 로드
  - `StageManager.showStage(n)` — n번 스테이지 렌더링
  - `StageManager.handleChoice(stage, choice)` — A/B 선택 처리
  - `StageManager.showResult()` — 최종 리포트 화면 전환
  - `StageManager.reset()` — 재도전 초기화
- **상태 관리:** 선택 이력, 점수, 현재 스테이지 index를 단일 `state` 객체로 관리
- **화면 전환:** CSS class 토글 방식 (`display:none` 금지 → `opacity + transform` 트랜지션)

### 5.2 AudioEngine.js — 오디오 엔진

- **브라우저 자동재생 정책 대응:** 인트로 화면 최초 탭/클릭 시 `AudioContext` 초기화 (사용자 제스처 필요)
- **iOS Safari 대응:** `playsinline` 속성 + `muted` 초기화 후 unmute 패턴
- **음성 재생 흐름:**
  1. 스테이지 진입 → `AudioEngine.playVoice(audioSrc)` 호출
  2. 재생 중 음파 파형 SVG 애니메이션 연동 (`onPlay` 콜백)
  3. 재생 완료 → 선택 버튼 활성화 (재생 전 버튼 비활성 처리)
- **SFX 재생:** `AudioEngine.playSFX(sfxSrc)` — 싱크홀 효과음 트리거
- **폴백:** 오디오 로드 실패 시 텍스트 스크립트로 대체 표시

### 5.3 SinkholeEffect.js — 싱크홀 애니메이션

- **A 선택 시 트리거:** 화면 전체 소용돌이 + 빨려 들어가는 효과
- **구현 방식 (우선순위):**
  1. `[1순위]` CSS `@keyframes` — rotate + scale(0) + blur 조합으로 텍스트 왜곡
  2. `[2순위]` SVG filter `feTurbulence + feDisplacementMap` — 더 강렬한 왜곡
  3. `[3순위]` Canvas 소용돌이 — 저사양 기기 대응 폴백
- **애니메이션 시퀀스:**
  - `0.0~0.3s` — 화면 흔들림 (shake keyframe)
  - `0.3~0.8s` — 텍스트 소용돌이 + 빨간 글로우 확산
  - `0.8~1.2s` — scale(0)으로 수축하며 사라짐
  - `1.2~1.5s` — 결과 오버레이 fade-in
- **B 선택 시:** 초록 체크 파티클 + 간단 슬라이드 업 전환

### 5.4 ScoreTracker.js — 점수 및 리포트

- **선택 이력 누적:** 각 스테이지 완료 시 `{ stage, choice, result }` 객체 push
- **최종 리포트 생성:**
  - 통과 스테이지 수 기반 원형 게이지 애니메이션
  - 선택 이력별 맞춤 피드백 텍스트 렌더링
  - 틀린 선택 스테이지만 교육 카드 강조 표시
- **SNS 공유:** URL 파라미터로 결과값 인코딩하여 카카오/트위터 공유 링크 생성

### 6.2 모바일 최적화 기준

- **뷰포트:** `<meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1">`
- **터치 타겟:** 모든 버튼 최소 48×48px (iOS/Android HIG 기준)
- **폰트:** `clamp()` 함수로 뷰포트 반응형 크기 적용
- **레이아웃:** Flexbox 기반 단일 컬럼, `max-width: 480px` 중앙 정렬
- **Safe Area:** `env(safe-area-inset-*)` 대응 — iPhone 노치/Dynamic Island
- **브레이크포인트:** 375px (iPhone SE) / 390px (iPhone 14) / 414px (Plus) 기준

## 9. 바이브코딩 프롬프트 가이드

### 9.1 초기 세팅 프롬프트

```
다음 조건으로 보이스피싱크홀 이벤트 랜딩 페이지의 [파일명]을 작성해줘.
- 빌드 도구 없는 순수 HTML/CSS/JS (ES6+, 모듈 X)
- 블랙(#0a0a0a) 배경, 빨간(#C0392B) 포인트 컬러 테마
- 모바일 우선 단일 컬럼 레이아웃 (max-width: 480px)
- CSS Custom Properties는 이미 variables.css에 정의됨
- 외부 라이브러리 사용 금지
- 한국어 폰트: Noto Sans KR (Google Fonts CDN)
```

### 9.2 싱크홀 애니메이션 프롬프트

```
SinkholeEffect.js를 작성해줘.
- A 선택 시 호출되는 triggerSinkhole(container) 함수 구현
- 애니메이션 시퀀스:
  ① shake (0~0.3s)
  → ② rotate+blur 왜곡 (0.3~0.8s)
  → ③ scale(0) 수축 (0.8~1.2s)
  → ④ 결과 오버레이 fade-in (1.2~1.5s)
- CSS @keyframes만 사용 (Canvas, WebGL 금지)
- will-change: transform 으로 GPU 가속 활성화
- 완료 후 Promise resolve 반환 (await 가능하게)
```

### 9.3 오디오 엔진 프롬프트

```
AudioEngine.js를 작성해줘.
- Web Audio API 기반, iOS Safari 자동재생 정책 완전 대응
- unlockAudio(): 사용자 제스처로 AudioContext resume
- playVoice(src, onProgress): 음성 재생 + 진행률 콜백
- playSFX(src): 효과음 즉시 재생 (겹침 가능)
- stop(): 현재 재생 중지
- 오디오 로드 실패 시 콘솔 경고 후 무음 진행 (게임 중단 없이)
```

### 9.4 StageManager 프롬프트

```
StageManager.js를 작성해줘.
- stages.js의 STAGES 배열 데이터를 기반으로 동작
- state 객체: { currentStage, history: [], score }
- showStage(n): 스테이지 데이터를 DOM에 렌더링
- handleChoice(choice): 'A' 또는 'B' 처리
  - 'sink' 결과 → SinkholeEffect.triggerSinkhole() await
  - 'safe' 결과 → 안전 피드백 애니메이션
  - 두 경우 모두 1.5초 후 자동으로 다음 스테이지 진행
- showResult(): 최종 리포트 화면 렌더링
- reset(): state 초기화 후 Stage 1 재시작
```
